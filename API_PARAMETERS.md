### 📝 사용자 제공 시 정확도 ↑ 가능 (선택)

다음 API들은 **검색 파라미터 명세가 공식 문서에 없음**. 마이페이지 "참고문서" `.docx` 파일을 다운로드해 전달하시면 정확한 파라미터 적용 가능:

| NO | 참고문서 파일 | 효과 |
|----|--------------|------|
| **142** | `IROS_402_의약품_국가출하승인정보_v1.0.docx` | 국가출하승인 검색 가능 → 광동제약 1만건 → 정확 카운트 |
| **144** | `IROS_403_의약품등_업체허가현황_v1.0.docx` | 업체 검색 가능 |
| **132** | `IROS_408_의약품GMP적합판정서발급현황_v1.0.docx` | 이미 검증 완료 (BSSH_NM) |
| **483** | `IROS_76_원료의약품(DMF)현황_v1.1.docx` | 원료의약품 검색 가능 |
| **539** | `IROS_50_의약품행정처분서비스_v1.6.docx` | 회수·판매중지 검색 가능 (있다면) |
| **534** | `IROS_73_의약품생산수입공급중단정보_v1.2.docx` | 공급중단 검색 가능 |
| **561** | (참고문서 미수록) | 특허 — 파라미터 없으면 클라이언트 필터링 유지 |

**없어도 운영 가능**: 클라이언트 사이드 필터링이 작동하므로 베니톨정 같은 정확한 단일 제품 조회는 이미 잘 동작합니다.
**있으면 좋은 효과**: 전체 데이터(특허 12만 건·재심사 11만 건 등)를 매번 받지 않아 응답 속도 ↑ + 호출 한도 절약.

### 📂 .docx 자료 전달 방법

1. https://www.data.go.kr/iim/api/selectAcountList.do (마이페이지 > 개발계정)
2. 각 API "상세보기" → "기본정보 > 참고문서" 클릭하여 .docx 다운로드
3. 다음 위치에 저장 후 Claude에게 알림:
   ```
   C:\Users\user\Desktop\Coding\cusor\drug_integrated_search\docs\refs\
   ```
4. Claude가 .docx 명세 읽고 [api_extras.py](api_extras.py) 의 파라미터 정정
